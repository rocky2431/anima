# Task 14: 实现四层记忆完整流程 (每日萃取 + 向量检索 + 记忆管理)

> **Status**: pending | **Priority**: P1 | **Complexity**: 6

## Context
**What**: 完善四层记忆架构: Layer 1 工作记忆 (内存) + Layer 2 实时记忆 (LanceDB) + Layer 3 结构化 (SQLite) + 每日记忆萃取流程 + 向量语义检索
**Why**: "有记忆"是 Anima 核心差异——角色能记住用户偏好、习惯、重要日期 (F6, Scenario 2)
**Constraints**:
- 每日萃取在 23:00 或关机时触发
- 重要性评分 > 7/10 的事件才存入长期记忆
- 向量检索 top-k 相关度 >85%
- 用户可查看/编辑/删除任何记忆

## Implementation
**Target Files**:
- `packages/persona-engine/src/memory-orchestrator.ts` (create) — 三层认知记忆编排
- `packages/context-engine/src/storage/memory-extractor.ts` (create) — 每日萃取: LLM 提取重要事件+去重+合并
- `packages/context-engine/src/storage/vector-store.ts` (modify: 添加语义检索 API)
- `packages/context-engine/src/storage/document-store.ts` (modify: 添加用户画像/关系/重要日期表)

**Pattern**: 参考 INTEGRATION_PLAN.md 八、分层记忆与存储方案 + architecture.md §6.3 每日萃取流程
**Tech Notes**: LLM 萃取用 AI SDK 6 generateObject; embedding 用 @xsai/embed 或 AI SDK embed; LanceDB search 支持 cosine similarity
