# Review Coordinator Memory

## Deduplication Patterns

### High-overlap areas
- **Error handling in catch blocks**: review-code, review-errors, review-tests, and review-comments frequently all flag the same catch block. Expect 3-4 agents to report on a single bare catch. The error-handling category from review-code and review-errors will merge; test-quality and comments categories will remain separate findings.
- **Integration/orphan findings**: review-code is the primary reporter. Rarely duplicated by other agents.

### Merge rules learned
- When merging error-handling duplicates on the same catch block, the review-errors agent typically has the highest confidence (99) and most specific description. Prefer its description.
- review-comments findings on the same lines as error-handling findings are NOT duplicates (different category). They remain separate.
- review-errors-002 (line 95, return false in catch) merges with review-errors-001 (line 92, the catch itself) when both are error-handling on the same file within +-3 lines.
